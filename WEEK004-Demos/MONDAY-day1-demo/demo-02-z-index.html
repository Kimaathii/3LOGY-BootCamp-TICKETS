4. Stacking Context (Advanced)

Important: A parent's z-index creates a stacking context.

Children can't escape parent's z-index level!

Parent 1 (z-index: 1)
Child with z-index: 9999
(Still below Parent 2!)
Parent 2 (z-index: 2)
Child (no z-index)
(Appears above Parent 1's children!)

📝 Z-Index Rules

  1. Requires Positioning: Z-index only works on positioned elements (relative, absolute, fixed, sticky)
  2. Higher = On Top: Higher z-index values appear on top
  3. Stacking Context: Parent z-index creates boundary - children can't escape
  4. Default: Elements have z-index: auto (same as 0)
  5. Common Values: Tooltips (100), Dropdowns (1000), Modals (10000)

🎯 Best Practices

  • Use scale: 1, 10, 100, 1000 (not 1, 2, 3, 4...)
  • Document z-index values in CSS comments
  • Avoid z-index wars (999999 is a code smell!)
  • Use CSS custom properties for z-index values
