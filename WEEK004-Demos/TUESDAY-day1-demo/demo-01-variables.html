🎯 Demo 01: Variables & Data Types

Understanding how to store and manipulate data in JavaScript

👉 Open your browser console (F12) and follow along!
All examples below are also running in the console.

1. Variable Declaration: var, let, const

var - Old way (avoid in modern JavaScript):

var oldWay = "I'm a var";
// Function-scoped, can cause bugs

let - Modern way for variables that change:

let age = 25;
age = 26; // Can be reassigned
console.log(age); // 26
Output:

const - For constants (can't be reassigned):

const PI = 3.14159;
// PI = 3.14; // ❌ Error: Assignment to constant variable
✅ Best Practice:
Use const by default. Only use let when you know the value will change. Avoid var!

2. Primitive Data Types

JavaScript has 7 primitive data types:

Type Example Description
String "Hello", 'World' Text data
Number 42, 3.14 Integers and floats
Boolean true, false True or false
Undefined undefined Variable declared but not assigned
Null null Intentional absence of value
Symbol Symbol() Unique identifier (advanced)
BigInt 9007199254740991n Very large integers (advanced)

String Examples:

const firstName = "Alice";
const lastName = 'Smith'; // Single or double quotes both work
const greeting = `Hello, ${firstName}!`; // Template literal (backticks)
console.log(greeting);

Number Examples:

const integer = 42;
const float = 3.14;
const negative = -10;
const sum = integer + float;
console.log(`Sum: ${sum}`); // 45.14
Output:

Boolean Examples:

const isStudent = true;
const hasGraduated = false;
const isEnrolled = 10 > 5; // true

3. The typeof Operator

Use typeof to check data type:

console.log(typeof "Hello"); // "string"
console.log(typeof 42); // "number"
console.log(typeof true); // "boolean"
console.log(typeof undefined); // "undefined"
console.log(typeof null); // "object" (this is a bug!)
console.log(typeof {}); // "object"
console.log(typeof []); // "object" (arrays are objects)
console.log(typeof function(){}); // "function"
Output:
⚠️ Common Gotcha:
typeof null returns "object", which is a famous JavaScript bug that can't be fixed (would break existing code). To check for null, use value === null.

4. Template Literals (Template Strings)

Use backticks (`) for string interpolation and multi-line strings:

const name = "Bob";
const age = 30;

// Old way (string concatenation):
const oldWay = "My name is " + name + " and I'm " + age + " years old.";

// New way (template literal):
const newWay = `My name is ${name} and I'm ${age} years old.`;

console.log(newWay);
Output:

Multi-line Strings:

const poem = `Roses are red,
Violets are blue,
JavaScript is awesome,
And so are you!`;

console.log(poem);
Output:

Expressions in Template Literals:

const price = 19.99;
const quantity = 3;

console.log(`Total: $${(price * quantity).toFixed(2)}`); // Total: $59.97
Output:

5. Type Coercion (Automatic Type Conversion)

JavaScript automatically converts types in some situations:

// Number + String = String concatenation
console.log(5 + "5"); // "55" (string)

// String - Number = Number subtraction
console.log("10" - 5); // 5 (number)

// Boolean to Number
console.log(true + 1); // 2 (true becomes 1)
console.log(false + 1); // 1 (false becomes 0)
Output:
⚠️ Be Careful:
Type coercion can cause unexpected bugs! Use explicit conversion when needed:
  • Number("42") - convert to number
  • String(42) - convert to string
  • Boolean(1) - convert to boolean
