🎯 Demo 04: Event Listeners

Making your page respond to user actions — clicks, typing, hovering

1. Click Events — addEventListener('click')

const button = document.querySelector('#myButton'); button.addEventListener('click', function() { console.log('Button was clicked!'); }); // The function runs EVERY TIME the button is clicked // "click" is the event type // The function is the "event handler" (what to do)

2. Mouse Events — hover, enter, leave

// Different mouse events: element.addEventListener('mouseenter', handler); // Mouse enters element element.addEventListener('mouseleave', handler); // Mouse leaves element element.addEventListener('mousemove', handler); // Mouse moves over element element.addEventListener('dblclick', handler); // Double click

3. Input Events — Live Typing Detection

const input = document.querySelector('#searchBox'); // 'input' event fires on EVERY keystroke input.addEventListener('input', function() { console.log('You typed:', input.value); // Perfect for live search, character counters, validation }); // 'focus' — when user clicks into the input // 'blur' — when user clicks away from input

4. The Event Object (e)

// The event handler receives an "event object" (e) button.addEventListener('click', function(e) { console.log(e.target); // Which element was clicked console.log(e.type); // "click" console.log(e.clientX); // Mouse X position console.log(e.clientY); // Mouse Y position }); // e.target is VERY useful — tells you exactly what was clicked!

✅ Best Practice — Event Delegation:
Instead of adding an event listener to EACH button, add ONE to the parent and use e.target to know which child was clicked. More efficient!

5. Keyboard Events

// Listen for key presses on the whole document document.addEventListener('keydown', function(e) { console.log('Key pressed:', e.key); console.log('Key code:', e.code); if (e.key === 'Enter') { console.log('Enter was pressed!'); } });
