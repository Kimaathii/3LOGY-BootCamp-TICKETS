createElement, appendChild, remove — building dynamic content

1. Create → Configure → Append

// 1. CREATE const li = document.createElement('li'); // 2. CONFIGURE li.textContent = 'New item'; li.className = 'list-item'; // 3. APPEND document.querySelector('#list').appendChild(li);

    2. Adding Delete Buttons

    function addItemWithDelete(text) { const li = document.createElement('li'); li.textContent = text; // Create delete button const deleteBtn = document.createElement('button'); deleteBtn.textContent = 'Delete'; deleteBtn.addEventListener('click', function() { li.remove(); // Remove THIS list item }); li.appendChild(deleteBtn); list.appendChild(li); }
    ✅ Pattern: Each item creates its own delete button. The button's click handler removes its parent li element.

    3. Building Complex Elements

    // Creating a card with multiple children function createCard(title, description) { const card = document.createElement('div'); card.className = 'card'; const h3 = document.createElement('h3'); h3.textContent = title; const p = document.createElement('p'); p.textContent = description; card.appendChild(h3); card.appendChild(p); container.appendChild(card); }

    4. innerHTML vs createElement

    // innerHTML — quick but less control list.innerHTML += '<li>New item</li>'; // createElement — more code but better control const li = document.createElement('li'); li.textContent = 'New item'; list.appendChild(li); // ⚠️ innerHTML rebuilds ALL children (slow for large lists) // createElement adds ONE element (fast, keeps event listeners)
    💡 When to use which:
    • innerHTML: Quick static content, clearing children (el.innerHTML = '')
    • createElement: Dynamic elements with event listeners, complex structures
