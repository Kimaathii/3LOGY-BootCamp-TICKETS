Three Ways to Add CSS

Understanding your options (and which to choose!)

❌ Method 1: Inline CSS (DON'T USE THIS)

This paragraph is styled with inline CSS

Code: 
<p style="color: #d63031; font-size: 18px; font-weight: bold;">
  This paragraph is styled with inline CSS
</p>

Problems with Inline CSS:

  • ❌ Mixes HTML structure with CSS presentation
  • ❌ Not reusable (must copy-paste to every element)
  • ❌ Hard to maintain (change 100 buttons? Edit 100 places!)
  • ❌ Can't be cached by browser
  • ❌ Highest specificity (overrides everything)
  • ❌ Makes HTML messy and hard to read

When to use: NEVER in production code! (Only acceptable for quick email HTML)

⚠️ Method 2: Internal CSS (OK for Small Projects)

This paragraph is styled with INTERNAL CSS (in the <style> tag in the <head>)

Code in <head> section: 
<head>
  <style>
    .internal-example {
      color: green;
      font-size: 20px;
      padding: 15px;
      background-color: #e8f5e9;
      border-left: 4px solid green;
    }
  </style>
</head>
Code in <body> section: 
<p class="internal-example">
  This paragraph is styled with INTERNAL CSS
</p>

Pros of Internal CSS:

  • ✅ Separates CSS from HTML structure
  • ✅ Reusable within the same page
  • ✅ Better than inline CSS
  • ✅ All styles in one file (single HTML file)

Cons of Internal CSS:

  • ⚠️ Can't be reused across multiple pages
  • ⚠️ Page loads slower (CSS not cached)
  • ⚠️ Hard to maintain large websites

When to use: Single-page projects, quick prototypes, or email templates

✅ Method 3: External CSS (BEST PRACTICE)

How External CSS Works:

1. Create separate CSS file (styles.css):

/* styles.css */
.button {
  background-color: #3498db;
  color: white;
  padding: 10px 20px;
  border-radius: 5px;
}

2. Link CSS file in HTML <head>:

<head>
  <link rel="stylesheet" href="styles.css">
</head>

3. Use classes in HTML:

<button class="button">Click Me</button>
<button class="button">Another Button</button>

✅ Why External CSS is Best:

  • ✅ Reusable: One CSS file styles entire website
  • ✅ Maintainable: Change one line, update everywhere
  • ✅ Performance: Browser caches CSS file
  • ✅ Organized: Separation of concerns (HTML structure, CSS presentation)
  • ✅ Scalable: Easy to add new styles
  • ✅ Collaborative: Designers work on CSS, developers on HTML
  • ✅ Professional: Industry standard

🎯 ALWAYS USE EXTERNAL CSS for real projects!

📊 Quick Comparison

Method Syntax Reusable? Recommendation
Inline style="..." ❌ No ❌ Avoid
Internal <style> in <head> ⚠️ Same page only ⚠️ Sometimes
External <link rel="stylesheet"> ✅ Entire website ✅ BEST

🎨 Key Takeaway: Always use External CSS for professional projects!

Separation of concerns = Clean, maintainable code
