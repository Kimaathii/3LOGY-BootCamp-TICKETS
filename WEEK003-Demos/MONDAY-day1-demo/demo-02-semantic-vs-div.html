Comparison: Div Soup vs Semantic HTML

Instructor: Show both examples side-by-side. Ask students which is clearer.

❌ Div Soup (Bad Practice)

<div class="header">
  <div class="title">My Website</div>
  <div class="nav">
    <div class="menu-item">Home</div>
    <div class="menu-item">About</div>
  </div>
</div>

<div class="main-content">
  <div class="article">
    <div class="article-title">Welcome</div>
    <div class="article-text">Content here</div>
  </div>
</div>

<div class="footer">
  <div class="copyright">© 2024</div>
</div>

Problems:

  • ❌ No semantic meaning
  • ❌ Screen readers can't navigate well
  • ❌ Poor SEO (search engines confused)
  • ❌ Relies on class names (can change)
  • ❌ Not self-documenting

✅ Semantic HTML (Best Practice)

<header>
  <h1>My Website</h1>
  <nav>
    <ul>
      <li><a href="/">Home</a></li>
      <li><a href="/about">About</a></li>
    </ul>
  </nav>
</header>

<main>
  <article>
    <h2>Welcome</h2>
    <p>Content here</p>
  </article>
</main>

<footer>
  <p>© 2024</p>
</footer>

Benefits:

  • ✅ Clear semantic meaning
  • ✅ Screen readers navigate easily
  • ✅ Better SEO (search engines understand)
  • ✅ Self-documenting code
  • ✅ Accessibility built-in

💡 Teaching Point

Ask students: "If you opened this code in 6 months, which version would be easier to understand?"

Emphasize: Semantic HTML is like writing with proper nouns instead of just "thing" - it's clear what everything is!
