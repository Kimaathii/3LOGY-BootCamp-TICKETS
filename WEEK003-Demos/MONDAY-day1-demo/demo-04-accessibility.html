Accessibility Examples: Good vs Bad Practices

Instructor: Demonstrate each example. Use Tab key to navigate, show focus states.

1. Image Alt Text

❌ Bad: Missing or useless alt text

<img src="sunset.jpg">
<img src="graph.jpg" alt="image">
<img src="profile.jpg" alt="picture of me">

Problem: Screen reader says nothing or generic "image"

✅ Good: Descriptive alt text

<img src="sunset.jpg" alt="Golden sunset over mountain range with purple sky">
<img src="graph.jpg" alt="Bar chart showing 50% increase in sales from 2023 to 2024">
<img src="profile.jpg" alt="Jane Smith, web developer, smiling in office">

Tip: Describe what's in the image concisely

2. Form Labels

❌ Bad: Missing labels or placeholder-only

<input type="text" placeholder="Enter your name">
  • ❌ Placeholder disappears when typing
  • ❌ Screen reader doesn't know purpose
  • ❌ Can't click to focus

✅ Good: Proper label with for attribute

<label for="username">Name:</label>
<input type="text" id="username" name="username" placeholder="John Doe">
  • ✅ Label always visible
  • ✅ Click label to focus input
  • ✅ Screen reader announces "Name, edit text"

Try it: Click the "Name:" label above!

3. Heading Hierarchy

❌ Bad: Skipping levels or using for styling

<h1>Page Title</h1>
<h3>Subsection</h3>  <!-- Skipped H2! -->
<h5>Bold text</h5>    <!-- Used for styling, not structure -->

Problems:

  • ❌ Confuses screen reader navigation
  • ❌ Breaks document outline
  • ❌ Poor SEO

✅ Good: Logical nesting

<h1>Page Title</h1>
  <h2>Section 1</h2>
    <h3>Subsection 1.1</h3>
    <h3>Subsection 1.2</h3>
  <h2>Section 2</h2>

Benefits:

  • ✅ Clear document outline
  • ✅ Screen readers can jump between headings
  • ✅ Better SEO

Rule: Only ONE H1 per page, don't skip levels

4. Keyboard Navigation

❌ Bad: Removed focus outline

<style>
  *:focus { outline: none; }
</style>

<a href="#">Link with no focus indicator</a>

Link with no focus (try Tab key)

Problem: Keyboard users can't see where they are!

✅ Good: Visible focus indicator

<style>
  a:focus {
    outline: 3px solid #007acc;
    outline-offset: 2px;
  }
</style>

<a href="#">Link with clear focus indicator</a>

Link with visible focus (try Tab key)

Try: Tab to this link and see the blue outline!

5. Color Contrast

❌ Bad: Low contrast (fails WCAG)

<p style="color: #ccc; background: #fff;">
  Light gray text on white background
</p>

This text is hard to read (1.6:1 contrast ratio)

Problems:

  • ❌ Hard to read for everyone
  • ❌ Impossible with low vision
  • ❌ Fails WCAG AA standard (needs 4.5:1)

✅ Good: High contrast (passes WCAG AA)

<p style="color: #333; background: #fff;">
  Dark gray text on white background
</p>

This text is easy to read (12.6:1 contrast ratio)

Benefits:

  • ✅ Easy to read for everyone
  • ✅ Works in bright sunlight
  • ✅ Passes WCAG AA and AAA

Tool: Chrome DevTools > Inspect > Accessibility tab

6. ARIA Labels

❌ Bad: Icon button with no label

<button>×</button>

Problem: Screen reader says "times button" - not helpful!

✅ Good: ARIA label describes purpose

<button aria-label="Close modal">×</button>

Benefit: Screen reader says "Close modal button" - clear!

✅ Best: Use semantic HTML first

<button>
  <span aria-hidden="true">×</span>
  Close
</button>

Best: Visible text is better than ARIA when possible
