📱 Responsive Navigation Demo

Try this:

  1. Resize your browser to mobile size (<768px)
  2. Click the hamburger menu (☰) in the top right
  3. See the slide-in navigation
  4. Resize to desktop (>768px)
  5. Watch the navigation transform to horizontal layout

🎯 How It Works

Mobile Approach (Default):

  • Navigation hidden off-screen (right: -100%)
  • Hamburger button visible
  • Click hamburger → add 'active' class → nav slides in
  • Dark overlay appears behind nav

Desktop Approach (768px+):

  • Navigation always visible
  • Horizontal flexbox layout
  • Hamburger button hidden
  • No overlay needed

💡 Key Techniques

1. Fixed Mobile Menu

nav { position: fixed; right: -100%; /* Hidden off-screen */ transition: right 0.3s ease; } nav.active { right: 0; /* Slide in when active */ }

2. Hamburger to X Animation

.hamburger.active span:nth-child(1) { transform: rotate(45deg) translateY(12px); } .hamburger.active span:nth-child(2) { opacity: 0; /* Hide middle bar */ } .hamburger.active span:nth-child(3) { transform: rotate(-45deg) translateY(-12px); }

3. Desktop Override

@media (min-width: 768px) { nav { position: static; /* Override fixed positioning */ width: auto; } .hamburger { display: none; /* Hide hamburger on desktop */ } }

✅ Best Practices

  • ✓ Always include aria-label on hamburger button
  • ✓ Make nav accessible via keyboard (Tab key)
  • ✓ Use CSS transitions for smooth animations
  • ✓ Add overlay to dim content behind mobile menu
  • ✓ Close menu when clicking overlay or link
  • ✓ Prevent body scroll when mobile menu is open
  • ✓ Use position: sticky for nav that stays visible on scroll
