🎯 Flex Item Properties - Individual Control

Container vs Item Properties

So far we've learned container properties (affect ALL items).
Now we learn item properties (affect INDIVIDUAL items).

Container Properties

Set on parent, affect ALL children

  • display: flex
  • justify-content
  • align-items
  • flex-direction
  • flex-wrap
  • gap

Item Properties

Set on children, affect ONE item

  • flex-grow
  • flex-shrink
  • flex-basis
  • flex (shorthand)
  • align-self

Part 2: flex-grow (How Items Expand)

flex-grow controls how items expand to fill extra space.

/* Make one item expand */
.item:nth-child(2) {
  flex-grow: 1;  /* This item takes extra space */
}

/* Equal width columns */
.item {
  flex-grow: 1;  /* All items share space equally */
}
🎓 Real-world use:
- Search bar that expands in navigation
- Main content area taking remaining space
- Equal-width columns in a grid

Part 3: flex-shrink (How Items Contract)

flex-shrink controls how items shrink when space is tight.
Default is 1 (can shrink). Set to 0 to prevent shrinking.

All items: flex-shrink: 1 (default - can shrink)

Item 1: flex-shrink: 0 (won't shrink)

Item 3
shrink: 1

/* Prevent one item from shrinking */
.sidebar {
  flex-shrink: 0;   /* Maintains width */
  min-width: 200px;
}

.main {
  flex-shrink: 1;   /* Can shrink (default) */
}
💡 Common pattern: Fixed sidebar that doesn't shrink, flexible main content that does.

Part 4: flex-basis (Initial Size)

flex-basis sets the initial size of an item before growing/shrinking.
Think of it as "starting width" (better than width because it respects flex-direction).

Different flex-basis values

100px basis
200px basis
100px basis
.item {
  flex-basis: 200px;  /* Start at 200px wide */
}

/* flex-basis vs width: */
/* width: Always 200px wide */
/* flex-basis: 200px starting point, can grow/shrink */
🎓 Key difference:
width: Fixed size
flex-basis: Starting size (can still grow/shrink with flex-grow/shrink)

Part 5: flex Shorthand ⭐ (Most Common)

flex is shorthand for: flex-grow flex-shrink flex-basis
This is what you'll use 90% of the time!

Common Patterns:

flex: 1;              /* Same as: flex: 1 1 0 */
                      /* = Grow, shrink, 0 basis */
                      /* = Equal width columns */

flex: 0 0 200px;      /* = Don't grow, don't shrink, 200px */
                      /* = Fixed width sidebar */

flex: 2;              /* Same as: flex: 2 1 0 */
                      /* = Grows 2× as fast */

flex: auto;           /* Same as: flex: 1 1 auto */
                      /* = Grow/shrink from content size */

Example: Equal width columns (flex: 1)

Example: Fixed + Flexible (sidebar pattern)

🎯 You'll mostly use:
flex: 1; for equal columns
flex: 0 0 200px; for fixed sidebars

Part 6: align-self (Override Alignment)

align-self overrides the container's align-items for ONE specific item.

🎯 Key Takeaways

  • flex-grow: How items expand (0 = don't grow, 1 = grow equally)
  • flex-shrink: How items contract (0 = don't shrink, 1 = can shrink)
  • flex-basis: Starting size before growing/shrinking
  • flex: 1; = Equal width columns (most common!)
  • flex: 0 0 200px; = Fixed 200px width (sidebars)
  • align-self: Override alignment for one item
🎓 For Instructor:
- Emphasize flex shorthand (this is what students will use most)
- Show sidebar pattern - used on TONS of websites
- Demonstrate flex: 1 vs flex: 2 (proportional growing)
- Have students inspect in DevTools to see computed values
- Remind: Container properties (all items) vs Item properties (one item)
